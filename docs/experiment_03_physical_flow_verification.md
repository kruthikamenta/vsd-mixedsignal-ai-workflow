# Experiment 3 - Physical Design Output Verification

## Objective

To verify the physical design outputs generated by the reference repository and understand the major stages of the OpenLane RTL-to-GDSII implementation flow.

## AI Tool Used

- ChatGPT

### Repository Output Verification

Command used:

```bash
tree openlane/results -L 2
```

Observation:

- All major implementation stages were present.
- Separate directories existed for synthesis, floorplanning, placement, routing, DRC, and layout outputs.
- The repository contained a complete RTL-to-GDSII implementation flow.

### Floorplan Verification

Command used:

```bash
head -20 openlane/results/floorplan/design_mux.floorplan.def
```

Observation:

- The floorplan DEF file was successfully generated.

- Core area information was available.
